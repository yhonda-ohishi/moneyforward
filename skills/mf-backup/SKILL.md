# MoneyForward 仕訳バックアップスキル

仕訳データを変更する前にバックアップを取る。これにより、誤った変更があっても元に戻せる。

## いつ使うか

以下の操作を行う**前**に、このスキルのバックアップ手順を実行する:

- `mfc_ca_putJournals`（仕訳の更新）
- `mfc_ca_postJournals`（仕訳の登録）
- `mfc_ca_postTransactions`（取引の登録）
- ユーザーが仕訳の修正・変更・削除を依頼したとき

バックアップは**変更セッションごとに1回**でよい。同じセッション内で複数の仕訳を変更する場合、最初の変更前に1回取ればOK。

## バックアップ手順

### 1. バックアップディレクトリの確認

バックアップは `c:\moneyforward\backups\` に保存する。

```
c:\moneyforward\backups\
  └── YYYY-MM-DD_HHMMSS\
      ├── journals.json       # 全仕訳データ
      └── backup_info.json    # バックアップのメタ情報
```

### 2. 全仕訳を取得

`mfc_ca_getJournals` で現在の会計期間の全仕訳を取得する。

- `mfc_ca_currentOffice` で会計期間を確認
- `per_page=100` で全ページ取得（ページネーションで漏れなく）
- 取得した生のJSONレスポンスをそのまま保存する（復元に使えるように）

### 3. スクリプトで保存

MCP結果が一時ファイルに保存された場合（大量データ時）、スクリプトで自動保存する:

```bash
python c:/moneyforward/skills/mf-backup/scripts/backup.py <一時ファイルパス> --reason "変更理由"
```

結果が直接返ってきた場合は、手動でjournals.jsonとbackup_info.jsonを作成してもよい。

### 4. ユーザーに報告

バックアップ完了後、以下を表示してから変更作業に進む:

```
バックアップ完了: c:\moneyforward\backups\{timestamp}\
  仕訳数: {件数}件
  理由: {変更の概要}
```

`reason` にはユーザーがこれから行う変更の概要を記載する。ユーザーに確認せずに推測で書いてよい。

## 復元について

ユーザーが「元に戻して」「バックアップから復元」と言った場合:

1. 最新のバックアップディレクトリから `journals.json` を読む
2. 変更した仕訳のIDを特定
3. バックアップデータの内容で `mfc_ca_putJournals` を使って復元
4. 復元前にも再度バックアップを取る（復元の復元ができるように）

## 注意事項

- バックアップはローカルファイルに保存するだけなので、MoneyForward側のデータには影響しない
- アクセストークンの有効期限（1時間）に注意。トークン切れの場合は再認証してからバックアップを取る
- バックアップファイルには個人情報・財務データが含まれるため、gitにコミットしない（.gitignoreに追加すること）
